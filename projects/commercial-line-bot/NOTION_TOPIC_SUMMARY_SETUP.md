# Notion 主題總結資料庫設置指引

## 📋 建立新資料庫

1. 在你的 Notion workspace 中，創建一個新的資料庫
2. 命名為：**主題總結** 或 **Topic Summaries**

---

## 🗂️ 資料庫欄位設置

請確保資料庫包含以下欄位（**欄位名稱必須完全一致**）：

### 必要欄位

| 欄位名稱 | 類型 | 說明 |
|---------|------|------|
| **標題** | Title | 主題總結的標題（例如：「客戶開發完整知識地圖」） |
| **主題標籤** | Select | 主題的標籤名稱（例如：客戶開發、定價策略） |
| **總結內容** | Rich Text | AI 生成的完整知識地圖（500-1000字） |
| **來源統計** | Number | 整合了幾個來源（知識片段 + 對話總結） |
| **最後更新日期** | Date | 總結最後更新的日期 |
| **狀態** | Select | 狀態選項：已完成、草稿、需更新 |

### 可選欄位（用於追蹤來源）

| 欄位名稱 | 類型 | 說明 |
|---------|------|------|
| **關聯知識片段** | Relation | 關聯到「知識片段」資料庫 |
| **關聯對話總結** | Relation | 關聯到「主要問題」資料庫 |

---

## ⚙️ 環境變數設置

1. 複製新建資料庫的 ID
   - 打開資料庫頁面
   - URL 格式：`https://notion.so/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
   - 複製 `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` 這部分

2. 在 Vercel 或本地 `.env` 檔案中新增：
   ```
   NOTION_TOPIC_SUMMARY_DB_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

3. 如果是 Vercel 部署：
   - 進入 Vercel 專案設定
   - 到 Environment Variables 分頁
   - 新增變數：`NOTION_TOPIC_SUMMARY_DB_ID`
   - 貼上資料庫 ID
   - **重新部署**專案

## 📊 狀態 Select 選項設置

在「狀態」欄位中，建立以下選項：

- **已完成** （主要狀態）
- **草稿** （未來可用）
- **需更新** （當來源內容變化時）

---

## ✅ 驗證設置

完成設置後，在 LINE 中測試：

1. **生成主題總結**：
   ```
   總結 客戶開發
   ```
   系統會自動抓取所有與「客戶開發」相關的知識片段和對話總結，生成完整知識地圖。

2. **查詢主題總結**：
   ```
   查詢 客戶開發
   ```
   應該會看到三層內容：
   - 【主題總結】✨ 客戶開發完整知識地圖
   - 【精煉洞察】📚 知識片段
   - 【對話總結】💡 歷史洞察

## 🔧 疑難排解

### 問題：執行「總結 客戶開發」後出現錯誤

**可能原因**：
1. 環境變數 `NOTION_TOPIC_SUMMARY_DB_ID` 未設置
2. Notion 資料庫欄位名稱不正確
3. Notion Integration 未授權此資料庫

**解決方法**：
1. 確認 `.env` 或 Vercel 環境變數中有 `NOTION_TOPIC_SUMMARY_DB_ID`
2. 檢查資料庫欄位名稱是否完全一致（包括空格和中文字）
3. 在 Notion 中，確保你的 Integration 有存取此資料庫的權限
   - 點擊資料庫右上角 `...`
   - 選擇 `Connections` → 新增你的 Integration

### 問題：「查詢」指令看不到主題總結

**可能原因**：
- 尚未為該標籤生成主題總結

**解決方法**：
- 先執行 `總結 [標籤名稱]` 生成主題總結
- 再執行 `查詢 [標籤名稱]` 就能看到

## 📖 使用流程

### 完整知識積累流程

```
1. 對話訓練
   ↓
2. 儲存知識片段（「儲存」指令）
   ↓
3. 累積 5-10 個片段後
   ↓
4. 生成主題總結（「總結 客戶開發」）
   ↓
5. 隨時查詢（「查詢 客戶開發」）
   → 看到完整知識地圖
```

### 更新主題總結

當你累積了新的知識片段或對話後：

```
總結 客戶開發
```

系統會自動：
- 檢查是否已有主題總結
- 抓取所有最新內容
- **更新**現有總結（而非創建新的）
- 更新來源統計和日期

---

## 🎯 最佳實踐

1. **定期更新**：每累積 3-5 個新知識片段，就重新生成主題總結
2. **標籤策略**：優先為高頻標籤生成主題總結（商業、個人成長類）
3. **內容管理**：在 Notion 中可以手動編輯 AI 生成的總結，補充個人化內容
4. **版本追蹤**：Notion 會自動保留編輯歷史，可以看到知識地圖的演進

## 📚 三層知識架構說明

| 層級 | 名稱 | 用途 | 數量 | 長度 |
|------|------|------|------|------|
| **L0** | 主題總結 | 系統化知識地圖 | 1個/主題 | 500-1000字 |
| **L1** | 知識片段 | 可複用方法論 | 多個 | 100-200字 |
| **L2** | 對話總結 | 含盲點的洞察 | 多個 | 50-100字 |

**設計理念**：
- L0 = Knowledge Anchor（知識錨點）
- L1 = Atomic Notes（原子筆記）
- L2 = Reflection Notes（反思筆記）

這是數位化的 Zettelkasten（卡片盒筆記法）！
